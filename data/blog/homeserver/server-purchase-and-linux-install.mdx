프로그래밍 공부를 본격적으로 시작했던 2020년 무렵, IT 인프라의 중심은 이미 온프레미스에서 클라우드로 넘어가 있었습니다. 자연스럽게 저 또한 각종 공모전이나 개인 프로젝트를 AWS 환경에서 진행하며 커리어를 시작했습니다.

실무에서도 코스콤 금융 클라우드, NCP, AWS, GCP 등 다양한 클라우드 플랫폼을 경험했습니다. 손쉽게 인프라가 생성되는 클라우드 환경이 훨씬 익숙하고 편리했지만, 개인 프로젝트를 위한 인프라만큼은 굳이 '홈서버'로 구축하기로 결심했습니다. 그 이유는 크게 세 가지입니다.

### 1. 비용 스트레스에서의 해방
가장 현실적인 이유는 역시 '비용'입니다. AWS EC2 등을 24시간 365일 구동한다고 가정했을 때, 원하는 사양을 맞추려면 매달 지불해야 하는 고정 비용이 꽤 부담스럽습니다. 스팟 인스턴스, 서버리스서비스, 크레딧등을 활용하며 비용 절감을 위한 다양한 방법이 있겠지만 취미로 운영하는 서버 때문에 비용 관리에 스트레스를 받고 싶지는 않았습니다. 홈서버는 초기 하드웨어 구입 비용이 비교적 높은 대신 이후에는 전기 요금만 지불하면되기 떼문에 장기적으로는 비용효율적이고 스트레스가 적은 선택지라고 판단했습니다.

### 2. 기술의 밑바닥부터 이해하는 경험
클라우드는 편리하지만 그만큼 많은 기술들이 추상화되어 숨겨져 있습니다. 저는 편리함 뒤에 가려진 이러한 영역들을 직접 다뤄보고 싶었습니다. 베어메탈 환경에서 OS를 설치하는 것부터 네트워크를 직접 구성해보는 과정이 인프라에 대한 이해도를 높여줄 수 있을 것이라고 생각했습니다.

### 3. 온전한 소유가 주는 만족감
사실 가장 큰 이유는 단순한 소유욕 때문인 것 같습니다. 클라우드 상의 가상 머신이 아닌 물리적인 서버를 직접 소유하고 관리하고 싶었습니다. 내 공간에 존재하는 나만의 서버를 갖는다는 점이 구축을 결심한 가장 솔직한 이유입니다.


# 구입 목록

홈서버를 구축하기 위해 구입한 하드웨어 목록입니다. 초기 비용이 조금 들더라도 오랫동안 사용할 목적으로 CPU 성능과 램 용량을 넉넉하게 구성했습니다.

### 1. Mini PC
![Mini PC Purchase](/static/images/blog/homeserver/minipc_purchase.png)
서버의 핵심이 되는 본체는 공간 효율성과 전력 소모를 고려하여 미니 PC로 결정했습니다.

CPU는 **AMD Ryzen™ 7 7735HS** 프로세서가 탑재된 모델을 선택했습니다. 구입 당시 여러 제품을 비교해본 결과, 성능 대비 가격(가성비)이 가장 훌륭하다고 판단했습니다.

### 2. RAM
![RAM Purchase](/static/images/blog/homeserver/ram_purchase.png)
여러 어플리케이션을 실행할 것을 고려해 32GB로 넉넉하게 구성하고 싶었지만 구입 당시 치솟은 램 가격이 발목을 잡았습니다. 결국 '일단 16GB로 버티다 가격이 안정되면 증설하자'는 계획으로 타협했으나 야속하게도 구입 시점(2025년 11월)보다 현재(2026년 2월) 가격이 2배나 뛰어버렸습니다. 재테크에는 영 소질이 없나 봅니다.

### 3. SSD
![SSD Purchase](/static/images/blog/homeserver/ssd_purchase.png)
SSD 역시 가격 폭등 때문에 가장 저렴한 브랜드의 512GB 모델을 선택하며 '나중에 램과 같이 증설하자'고 다짐했지만 이 또한 구입 시점(2025년 11월)보다 현재(2026년 2월) 가격이 3배 가까이 올라버렸습니다. 추 후 하드디스크(HDD)를 추가하는 방향으로 선회할 예정입니다.<br/><br/><br/>

# 조립 및 OS 설치

### 조립
조립 과정 자체는 매우 간단합니다. 미니 PC의 하판을 열고 준비한 RAM과 SSD를 슬롯에 장착하기만 하면 됩니다. 다만 제가 구입한 모델은 나사를 모두 제거했음에도 하판이 단단하게 고정되어 있어 분리가 쉽지 않았습니다. 결국 드라이버를 틈새에 조심스럽게 끼워 넣어 힘을 준 뒤에야 열 수 있었습니다.


**조립 전**
![Mini PC Open](/static/images/blog/homeserver/minipc_withour_ram_ssd.png)
하판을 분리하면 RAM과 NVMe SSD를 장착할 수 있는 슬롯이 보입니다.


**조립 후**
![Mini PC Assembled](/static/images/blog/homeserver/minipc_with_ram_ssd.png)
RAM 2개와 NVMe SSD를 장착 완료한 모습입니다. 별도의 도구 없이 손쉽게 장착할 수 있었습니다.

### OS 설치
OS는 **Ubuntu Server 24.04 LTS**를 사용했습니다. USB 부팅 디스크를 만들어 설치를 진행했습니다.
```bash
Distributor ID:	Ubuntu
Description:	Ubuntu 24.04.3 LTS
Release:	24.04
Codename:	noble
```


# 후기
이렇게 하드웨어 조립부터 우분투 서버 설치까지 홈서버 구축을 위한 첫 단추를 끼웠습니다.

글을 작성하고 있는 지금(2026년 2월)은 서버를 구입한 지 약 3개월이 지났습니다. 치솟은 부품 가격을 보고 있자니 '그때 눈 딱 감고 32GB 램과 1TB SSD를 질렀어야 했는데...' 하는 뒤늦은 후회가 밀려옵니다. 아무래도 돈 벌 팔자는 따로 있나 봅니다.

어쨌든 Mini PC에 리눅스까지 설치했지만 아직은 리눅스가 설치된 작은 컴퓨터에 불과합니다. 진정한 홈서버로 거듭나려면 집 밖에서도 접속할 수 있어야겠죠. 다음 글에서는 공유기 설정을 통해 **포트포워딩**과 **DDNS**를 구성하여, 외부에서도 언제든 내 서버에 접속할 수 있도록 만드는 과정을 다루겠습니다.
